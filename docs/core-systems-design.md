# 《开发商》核心系统设计文档

> 最后更新：2026-04-01

本文档描述游戏的四大核心数值系统：品质、竞品、市场热度、结算。这四个系统相互咬合，共同支撑"搜打撤"的核心决策循环。

---

## 一、品质系统

### 概述

品质是玩家的"产出物"——你做的游戏有多好。它决定了结算时的收入水平。

### 品质等级

| 等级 | 名称 | 分数区间 |
|------|------|---------|
| ROUGH | 粗糙 | 0 ~ 25 |
| ACCEPTABLE | 合格 | 25 ~ 50 |
| EXCELLENT | 精良 | 50 ~ 75 |
| MASTERPIECE | 杰作 | 75 ~ 100 |

### 品质积累

每消耗1个月研发时间，品质增加：

```
品质增量 = QUALITY_PER_MONTH × 外包效率倍率
```

| 参数 | 值 |
|------|-----|
| QUALITY_PER_MONTH | 2.2 |
| 外包效率（低/大学生兼职组） | ×1.0 |
| 外包效率（中/外包铁军） | ×1.5 |
| 外包效率（高/越南闪电队） | ×2.0 |

**实际增速**：2.2 / 3.3 / 4.4 点/月。

### 品质上限

品质不会超过主创等级对应的上限：

| 主创等级 | 品质上限 |
|---------|---------|
| 低（独立游戏愤青） | 40 |
| 中（鹅皇互娱主创） | 70 |
| 高（天命堂制作人） | 100 |

### 模糊显示机制

玩家看到的品质等级**不是**真实值：

- **内测前**：显示值 = 真实值 + 偏移量（每局固定，范围 ±10），可能看到错误的等级
- **内测后**：显示真实等级，偏移量被清除

这意味着花3个月做内测不是浪费——它消除了品质判断的不确定性，直接影响上线决策质量。

### 品质变动来源

| 来源 | 效果 |
|------|------|
| 研发推进（每月自动） | +2.2 ~ 4.4（受外包等级影响） |
| 空地格子 | +0.5 ~ 1.5（随机） |
| 宝箱格子 | +3.0 |
| 搜类事件 | +不等（事件定义） |
| 打磨成功 | +8.0 |
| 打磨失败放弃修复 | -25.0 |
| 打类事件保守结果 | -5 ~ -8（事件定义） |
| 打类事件冒险成功 | +3 ~ +5（事件定义） |

## 二、AI竞品系统

### 概述

每局有 2~3 个AI竞品与玩家争夺同题材用户。竞品的存在制造了时间压力——早上线能独占市场，但品质可能不够。

### 竞品生成

选题确认后，从三种性格中不重复抽取 2~3 个：

| 性格 | 品质范围 | 上线时机（占总月数比例） | 特殊行为 |
|------|---------|---------------------|---------|
| **激进派** | 20 ~ 45 | 15% ~ 35%（月5~12） | 无，到时间就上 |
| **保守派** | 55 ~ 80 | 65% ~ 85%（月23~30） | 无，到时间就上 |
| **跟风派** | 35 ~ 60 | 35% ~ 65%（月12~23） | 热度 ≥60 才上线 |

### 竞品上线触发

每月检查一次：

1. **跟风派特殊判定**：市场热度 ≥ 60 且已过最早上线时间 → 立即上线
2. **通用判定**：当前月 ≥ 预设上线月 → 上线

### 竞品上线的影响

竞品上线会**消耗市场热度**：

```
热度消耗 = 5.0 + (竞品品质 / 100) × 10.0
```

品质越高的竞品上线，对市场热度的打击越大（5~15点）。

### 玩家可感知信息

竞品的具体数值对玩家不可见，只提供模糊暗示：

| 信息 | 精确值 | 玩家看到的 |
|------|--------|-----------|
| 性格 | AGGRESSIVE | "他们动作很快" / "听说在赶工" |
| 性格 | CONSERVATIVE | "他们在憋大招" / "一直没动静" |
| 性格 | FOLLOWER | "听说在观望市场" / "跟风的味道很浓" |
| 品质 | 具体分值 | "可能是精品" / "品质一般"（±10随机偏差） |

## 三、市场热度系统

### 概述

市场热度代表玩家所选题材当前的市场关注度。热度高意味着用户多但竞品也可能扎堆，热度低意味着用户少但竞争也少。

### 热度计算

```
实际热度 = 基础热度 + 噪声 - 竞品累计消耗
```

**基础热度**（正弦波）：
```
base_heat = 50 + amplitude × sin((phase_offset + elapsed_months) × 2π / 24)
```

- 周期：24个月（游戏总时长36个月的2/3，保证一局内能看到完整的起伏）
- 基准线：50
- 振幅：由题材配置决定（每个题材不同）

**噪声**：确定性伪噪声，振幅 ±10，让曲线不那么"数学感"。

**竞品消耗**：每个竞品上线永久扣减 5~15 热度（不可恢复）。

### 玩家可感知的热度

玩家看不到精确数值，只看到模糊文字（每3个月刷新一次，有 ±5 感知误差）：

| 热度区间 | 显示文字 | 颜色 |
|---------|---------|------|
| 0 ~ 20 | 无人问津 | 灰色 |
| 20 ~ 40 | 逐渐升温 | 绿色 |
| 40 ~ 60 | 热度攀升中 | 黄色 |
| 60 ~ 80 | 市场火爆 | 橙色 |
| 80 ~ 100 | 全民狂热 | 红色 |

### 热度对结算的影响

热度通过**热度乘数**直接影响收入：

| 热度区间 | 收入乘数 |
|---------|---------|
| 0 ~ 20 | ×0.4 |
| 20 ~ 40 | ×0.7 |
| 40 ~ 60 | ×1.0 |
| 60 ~ 80 | ×1.3 |
| 80 ~ 100 | ×1.6 |

**最高与最低之间有4倍差距**——热度是收入的最大单一影响因子。

## 四、结算系统

### 概述

结算是游戏的"开奖时刻"。从玩家上线月到第36月，逐月计算收入，最终得出总收益。

### 结算参数

| 参数 | 范围 | 说明 |
|------|------|------|
| 用户池总量 | 240万 ~ 480万 | 每局随机 |
| 付费能力 | 0.6 ~ 1.25 | 每局随机 |
| 市场热度 | 0 ~ 100 | 上线时的实际热度 |
| 玩家品质 | 0 ~ 100 | 真实品质值 |
| 竞品数据 | 2~3个 | 各自品质 + 上线月 |

### 运营期

```
运营月数 = 36 - 上线月
```

越早上线，运营期越长，能吃到更多月份的收入。

### 逐月收入计算

每个运营月独立计算：

```
月基础用户 = 用户池 / 运营月数 × 市场成熟度 × 热度乘数
```

**市场成熟度**——越晚的月份，可触达用户越多：
```
maturity = 0.45 + 0.55 × (month / 36)
```
最早上线只能触达 45% 用户，最晚上线能触达 100%。

### 独占期 vs 竞争期

每个月根据是否有竞品已上线，分两种计算方式：

**独占月**（无竞品已上线）：
```
月收入 = 月基础用户 × 付费能力 × 品质因子
```

**竞争月**（有竞品已上线）：
```
玩家份额 = 玩家品质 / (玩家品质 + Σ竞品品质)
月收入 = 月基础用户 × 玩家份额 × 付费能力
```

### 品质因子

品质因子影响独占期收入，是一条S型曲线：

| 品质 | 品质因子 | 说明 |
|------|---------|------|
| 0 | 0.15 | 保底（再烂也有15%收入） |
| 20 | 0.50 | 低品质 |
| 40（基准线） | 1.00 | 基准 |
| 70 | 1.17 | 对数增长，收益递减 |
| 100 | 1.30 | 天花板 |

公式细节：
- 品质 < 基准线（40）：平滑S曲线 `floor + (1-floor) × smoothstep(quality/baseline)`
- 品质 ≥ 基准线（40）：对数增长 `1.0 + log(1 + excess/baseline) × 0.30`

### 总收入

```
总收入 = 独占期总收入 + 竞争期总收入
```

### 失败结算

时间耗尽未连通撤离点 = 游戏失败：
- 入场资源全部损失
- 无任何收入
- 金钱回到保底值（30万）

## 五、四大系统的咬合关系

```
入场选择（资源配置）
  │
  ├─ 主创等级 ──→ 品质上限
  ├─ 外包等级 ──→ 品质增速
  └─ 商务等级 ──→ 小游戏精力

选题
  │
  └─ 题材 ──→ 市场热度曲线 + AI竞品

研发过程
  │
  ├─ 时间消耗 ──→ 品质积累
  ├─ 内测 ──→ 揭示真实品质（消耗3月）
  ├─ 打磨 ──→ 品质提升/下降（消耗3月 + 风险）
  └─ 事件 ──→ 品质波动

上线决策（核心博弈）
  │
  ├─ 品质够不够？（内测过的知道，没内测的在猜）
  ├─ 热度好不好？（只能看模糊文字）
  ├─ 竞品上了没？（早上线能独占，但品质可能不够）
  └─ 时间够不够？（继续研发还是见好就收）

结算
  │
  ├─ 独占期收入 = f(上线时机, 热度, 品质因子)
  └─ 竞争期收入 = f(品质份额, 热度, 用户池)
```

### 关键决策点对照

| 决策 | 收益 | 代价 | 不确定性 |
|------|------|------|---------|
| 高级主创 | 品质上限高 | 贵（60万） | 无 |
| 做内测 | 知道真实品质 | 3个月时间 | 无 |
| 做打磨 | 70%概率品质+8 | 3个月 + 30%概率出bug | 有 |
| 早上线 | 独占期长 | 品质可能不够 | 竞品何时上线未知 |
| 晚上线 | 品质更高 | 独占期短，可能没有 | 市场热度可能下降 |
| 绕路开亮格 | 可能是稀有+好奖励 | 多消耗3~5个月 | 稀有度已知，内容未知 |
