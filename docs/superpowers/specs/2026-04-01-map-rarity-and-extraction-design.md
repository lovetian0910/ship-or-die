# 迷雾地图改造：稀有度简化 + 撤离点机制

> 状态：已确认
> 日期：2026-04-01
> 涉及系统：FogMap、FogMapGenerator、MapCell、DevRunning、Config

---

## 一、改动概述

两项改动，目标一致：让迷雾地图探索从"随便点格子"变成"有方向、有惊喜、有决策"。

| 改动 | 核心变化 | 设计目的 |
|------|---------|---------|
| 稀有度显示简化 | 5档光晕 → 2档（暗/亮） | 增加揭示惊喜感，减少最优解倾向 |
| 撤离点机制 | 随时上线 → 必须连通撤离点 | 给探索一个方向，制造"贪婪与恐惧"的拉扯 |

---

## 二、稀有度显示简化

### 2.1 迷雾状态（FOGGY）视觉

| 视觉档位 | 对应稀有度 | 光晕表现 | 格子占比 |
|----------|-----------|---------|---------|
| 暗格 | 普通（COMMON）+ 优良（UNCOMMON） | 无光晕，纯迷雾底色 | ~70% |
| 亮格 | 稀有（RARE）+ 史诗（EPIC）+ 传说（LEGENDARY） | 统一暖白微光 `Color(0.9, 0.85, 0.7)`，alpha=0.20 | ~30% |

- 两档都显示 `?` 问号
- 不显示任何悬停提示（月份消耗、稀有度名称等）
- 光晕不区分具体稀有度颜色

### 2.2 Loading 动画

- 呼吸脉冲 + 进度环颜色统一使用白色，不提前泄露稀有度
- Loading时长仍按实际稀有度走（普通0.5s ~ 传说2.5s），时长本身隐含信息但不直接告知

### 2.3 揭示瞬间（REVEALED）的"开箱"表演

进度环完成后，执行揭示动画：

1. **颜色闪变**：背景色从白色瞬间切换为稀有度对应颜色
   - 普通：灰色 `Color(0.7, 0.7, 0.7)`
   - 优良：绿色 `Color(0.3, 0.8, 0.3)`
   - 稀有：蓝色 `Color(0.3, 0.5, 0.9)`
   - 史诗：紫色 `Color(0.6, 0.2, 0.8)`
   - 传说：金色 `Color(0.9, 0.7, 0.1)`
2. **图标显示**：同时显示格子类型图标（🔍搜/⚠️打/💎宝箱 等）
3. **传说级特殊表现**：额外放大脉冲（scale 1.0→1.15→1.0，0.3秒）

### 2.4 Config 变更

新增常量：
```
MAP_GLOW_COLOR: Color = Color(0.9, 0.85, 0.7)   # 亮格统一光晕颜色
MAP_GLOW_ALPHA: float = 0.20                      # 亮格光晕透明度
MAP_GLOW_RARITY_THRESHOLD: String = "RARE"        # 稀有及以上显示光晕
LEGENDARY_REVEAL_SCALE: float = 1.15              # 传说揭示脉冲缩放
LEGENDARY_REVEAL_DURATION: float = 0.3            # 传说揭示脉冲时长
```

删除/废弃：MapCell 中原有的按稀有度分5档 alpha 的逻辑。

## 三、撤离点机制

### 3.1 地图生成变更

| 项目 | 旧设计 | 新设计 |
|------|--------|--------|
| EXIT 数量 | 3个（按比例分配） | **固定1个** |
| EXIT 可见性 | 迷雾中隐藏 | **开局即可见**，穿透迷雾 |
| EXIT 位置约束 | 曼哈顿距离≥5 | 保持≥5不变 |
| 地图重置 | 每10格提示上线，选继续则重置 | **删除**，全程一张地图 |
| 上线按钮 | 始终可用 | **初始锁定**，连通后解锁 |

### 3.2 撤离点视觉

- EXIT 格子始终可见，不受 HIDDEN/FOGGY 状态影响
- 显示 🚀 图标 + 橙色边框 `Color(1.0, 0.6, 0.2)`，区别于普通格子
- 未连通时图标半透明（alpha=0.5），连通后变为全亮（alpha=1.0）

### 3.3 "连通"判定逻辑

**定义**：从起点到撤离点之间，存在一条全部为 REVEALED 状态的四方向相邻格子路径。

**触发时机**：每次揭示一个格子后，执行 BFS 检查起点→撤离点的 REVEALED 连通性。

**实现要点**：
- BFS 只遍历 `states[row][col] == CellState.REVEALED` 的格子
- 起点始终是 REVEALED（开局就揭示）
- 撤离点本身不需要是 REVEALED，只需要它的某个邻格是 REVEALED 即可（玩家开到撤离点旁边 = 到达）
- 连通成功时自动将撤离点标记为 REVEALED

### 3.4 连通后的行为

1. **撤离点自动揭示**，显示为完整的 EXIT 样式（🚀 + 橙色背景）
2. **弹出确认框**：
   - 标题："撤离点已到达！"
   - 选项A："上线发布" → 进入结算
   - 选项B："继续探索" → 关闭弹窗，继续开格子
3. **UI解锁常驻上线按钮**：选"继续探索"后，顶部出现常驻"上线"按钮，随时可点
4. **常驻按钮点击** → 直接进入结算（不再弹确认框，避免冗余）

### 3.5 时间耗尽处理

- 36个月用完但路径未连通 = **游戏失败**
- 保持现有失败逻辑：入场资源全损
- 事件日志提示相应失败文案

### 3.6 删除的代码/配置

| 删除项 | 位置 |
|--------|------|
| `MAP_LAUNCH_PROMPT_INTERVAL` | Config.gd |
| `MAP_CELL_RATIOS.exit` 比例 | Config.gd（EXIT改为固定1个） |
| `_reset_map()` | dev_running.gd |
| `_total_cells_revealed` | dev_running.gd |
| `_next_launch_prompt_at` | dev_running.gd |
| 跨地图计数器相关逻辑 | dev_running.gd |
| 10格上线提示弹窗逻辑 | dev_running.gd |

### 3.7 Config 变更

新增常量：
```
MAP_EXIT_COUNT: int = 1                            # EXIT固定1个
MAP_EXIT_ALWAYS_VISIBLE: bool = true               # EXIT开局可见
MAP_EXIT_BORDER_COLOR: Color = Color(1.0, 0.6, 0.2)  # 橙色边框
```

删除常量：
```
MAP_LAUNCH_PROMPT_INTERVAL                          # 不再有定期上线提示
FORCED_LAUNCH_QUALITY_PENALTY                       # 时间耗尽=失败，无需惩罚系数
```

## 四、两项改动的协同效果

改动前的玩家行为：看光晕亮度 → 永远先点最亮的格子 → 每10格被打断问"要不要上线" → 随便点个上线按钮

改动后的玩家行为：
1. **开局看到撤离点位置** → 有了方向
2. **朝撤离点开路**，沿途遇到亮格（稀有+）决定是否绕路去开 → 路径选择博弈
3. **揭示格子时才知道具体稀有度** → 每次开格都有惊喜/失望的情绪波动
4. **到达撤离点后**：现在安全了，但品质够吗？要不要继续探索周边？ → 贪婪vs恐惧
5. **时间在流逝**，继续探索可能来不及 → 撤离决策压力

这就是完整的"搜打撤"情绪循环：搜（探索积累）→ 打（沿途危机）→ 撤（到达撤离点后的贪婪抉择）。
