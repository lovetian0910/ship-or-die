# 《开发商》可玩 Demo 完整设计规格

> 状态：待审核
> 日期：2026-03-30
> 截止：2026-04-03（4天）
> 目标受众：资深游戏制作人（求职作品）

---

## 一、项目约束

### 核心红线
- 4天内出可玩 Demo，不可延期
- 优先级：核心循环可玩 > 数值手感 > 内容丰富度 > 美术表现
- 宁可砍功能不可拖工期

### 展示目标
面试官（资深制作人）玩10分钟后应该能判断：
1. 设计者理解搜打撤的情绪内核（贪婪与恐惧的拉扯）
2. 系统之间有设计上的咬合关系，不是功能堆砌
3. 工程实现干净，代码结构清晰

---

## 二、核心循环

```
入场（选购资源）
  ↓
选题（参考市场热度信号）
  ↓
研发流程（时间在流逝）
  ├─ 前期：固定探索点 + 0-1次被动危机
  ├─ 中期：固定探索点 + 随机探索点 + 1-2次被动危机 + 【内测验证节点】
  ├─ 后期：偶发随机探索点 + 0-1次被动危机 + 【临上线打磨节点】
  ↓
上线决策（撤离时刻）
  ↓
市场竞争结算
  ↓
带收益离场 / 撤离失败（资源全损）
```

### 单局时长：约10分钟
| 阶段 | 时长 | 说明 |
|------|------|------|
| 入场/选题 | 1-2分钟 | 选资源、选题材 |
| 研发流程 | 5-6分钟 | 核心游玩阶段，事件+探索+可选节点 |
| 上线+结算 | 2-3分钟 | 撤离决策+结果呈现 |

---

## 三、时间机制（生命值等效）

- 每局固定开发时间上限（Demo中建议300秒游戏内时间，加上事件决策的暂停时间，玩家实际体验约5-6分钟。UI上呈现为"开发周期"）
- 时间耗尽前未上线 = 撤离失败，入场资源全部损失
- 时间消耗来源：
  - 基础研发推进（自动消耗）
  - 探索节点（主动消耗）
  - 可选节点（主动消耗）
  - 危机事件选项（被动/主动消耗）
- 时间是唯一的衰减维度，不需要血量/饱食度等额外数值

## 四、入场资源系统

### 经济循环
- 局内收益（金钱）→ 局间传递 → 购买下一局入场资源
- 失败损失入场资源，但保底金钱保证永远能再入局
- 无局外养成（天赋点等），保持失去的真实代价

### 三类资源 × 三级

**品质上限（主创）—— 决定游戏能做多好**

| 等级 | 命名 | 品质上限加成 | 价格（参考） |
|------|------|------------|------------|
| 低 | 独立游戏愤青 | 基础上限 | 低 |
| 中 | 鹅皇互娱主创 | 中等上限 | 中 |
| 高 | 天命堂制作人 | 最高上限 | 高 |

**时间效率（外包）—— 决定研发速度**

| 等级 | 命名 | 效率加成 | 价格（参考） |
|------|------|---------|------------|
| 低 | 大学生兼职组 | 基础速度 | 低 |
| 中 | 外包铁军 | 中等加速 | 中 |
| 高 | 越南闪电队 | 最快速度 | 高 |

**事件应对（商务）—— 危机事件中解锁额外选项**

| 等级 | 命名 | 应对能力 | 价格（参考） |
|------|------|---------|------------|
| 低 | 实习商务 | 解锁部分稳妥选项 | 低 |
| 中 | 万能商务 | 解锁全部稳妥选项 | 中 |
| 高 | 前渠道教父 | 解锁全部选项含冒险选项 | 高 |

### 资源机制要点
- 每类只选一个等级，不可叠加（你只能雇一个主创、一支外包、一个商务）
- 对应塔科夫：枪械（品质上限）/ 药品（时间效率）/ 护甲（事件应对）
- 入场时的资源配置决定了本局的策略空间——全投品质还是分散投资

## 五、选题系统

### 题材池
Demo 阶段提供 3-4 个可选题材（具体名称使用虚构名，影射真实品类）。

建议题材：
| 虚构名 | 影射品类 | 特征 |
|--------|---------|------|
| 待定 | 待定 | 待定 |

> ⚠️ 题材具体名称在实现阶段确定，不影响核心机制。

### 选题决策
- 玩家看到各题材的模糊热度信号后选择
- 热门题材：用户池大，但AI竞品扎堆概率高
- 冷门题材：竞争少，但用户池小
- 选题不可更改，选了就是选了

### 自定义游戏名
- 选题后玩家为自己的游戏起一个名字（自由文本输入）
- 纯代入感设计，不参与任何数值计算
- 用于研发流程UI展示和结算呈现（如"你的《星际牧场》获得了xxx收益"）

## 六、研发流程与可选节点

### 研发推进机制
- 选题后自动进入研发，时间开始流逝
- 品质随时间推进自动积累，速率受"时间效率"资源影响
- 品质积累有上限，受"品质上限"资源约束
- 品质对玩家呈现为**模糊等级**（如"粗糙/合格/精良/杰作"），不显示精确数值

### 可选节点1：内测验证（中期，信息揭示型）

**时机：** 研发进度约50%时出现，玩家可选择进入或跳过。

**机制：**
- 消耗一段固定时间（约30-45秒研发时间）
- 揭示当前品质的**真实等级**（平时品质显示是模糊的，带误差）
- 不改变品质本身，纯信息价值

**博弈点：**
- 时间充裕时：信息有价值，帮助后续决策
- 时间紧张时："花30秒只为看一个数字，值吗？"
- 与市场热度系统呼应：一个看外部（市场），一个看内部（自身品质），两个不完全信息拼合辅助上线决策

**设计意图（面试展示）：** 信息本身作为资源的设计思路——对应搜打撤中"搜"的本质不是直接获得战力，而是获得决策信息。

### 可选节点2：临上线打磨（后期，风险增益型）

**时机：** 研发进度约85%时出现，紧贴上线决策窗口。

**机制：**
- 消耗一段时间（约30-45秒研发时间）
- 进入前明示两种可能结果及概率：
  - **70%概率：** 品质小幅提升（约1个模糊等级的1/3）
  - **30%概率：** 发现严重bug，玩家面临二次选择：
    - 花15-20秒修复 → 品质不变（白花了打磨+修复的时间）
    - 放弃修复直接上线 → 品质反降1个等级

**博弈点：**
- 做过内测的玩家：知道自己品质真实水平，能理性评估"小幅提升有没有意义"
- 没做内测的玩家：不知道自己品质到底够不够，只能凭感觉赌
- 时间剩余多：风险可控，值得试
- 时间剩余少：30%概率触发bug修复可能直接导致时间耗尽=全损

**两节点咬合（核心设计亮点）：**
中期是否做内测，直接影响后期打磨的决策质量。这构成了一条完整的信息链路：
- 搜（内测）→ 知（品质真实水平）→ 判（打磨是否值得）→ 撤（上线时机）
- 不搜 → 不知 → 盲赌 → 可能失误

## 七、随机事件系统

> 详细设计见 `2026-03-17-random-event-system-design.md`，此处仅列关键参数。

### 三种事件入口

| 入口 | 玩家能动性 | 类型 |
|------|-----------|------|
| 被动危机 | 无法回避 | 打（危机中藏机会） |
| 固定探索点 | 主动选择进不进 | 搜70% / 打30% |
| 随机探索点 | 主动选择，有时间窗口（过期消失） | 搜70% / 打30% |

### 单局事件总量

| 来源 | 数量 |
|------|------|
| 被动危机 | 2-4次 |
| 固定探索点 | 2次 |
| 随机探索点 | 0-2次 |
| **总计** | **4-8次** |

### 阶段分布

| 阶段 | 被动危机 | 探索节点 |
|------|---------|---------|
| 前期（~30%时间） | 0-1次 | 1个固定 |
| 中期（~40%时间） | 1-2次 | 1个固定 + 随机 |
| 后期（~30%时间） | 0-1次 | 偶发随机 |

### 事件选项结构（打类事件）

| 选项 | 需要条件 | 代价 | 收益 |
|------|---------|------|------|
| 保守（永远可选） | 无 | 确定惩罚 | 无 |
| 稳妥 | 可能需要商务资源 | 消耗事件应对资源 | 消除惩罚 |
| 冒险 | 需要资源+时间 | 资源+大量时间 | 消除惩罚+额外收益 |

### 核心原则
- 单个事件内代价和收益全部明示，无随机判定
- 不确定性来自全局状态（后面还有没有危机、市场热度会不会变、AI竞品何时上线）
- 事件文本影射现实、虚构名称、带行业黑色幽默

## 八、上线决策（撤离等效）

这是整个游戏的**核心博弈时刻**。

### 机制
- 研发流程中玩家随时可以按"上线"按钮
- 按下后不可撤回，进入市场竞争结算
- 如果时间耗尽仍未上线 = 撤离失败，资源全损

### 决策维度
玩家在做上线决策时需要综合考量：
1. **品质**：当前品质够不够（如果做了内测就知道，没做就是模糊的）
2. **时间**：还剩多少时间，够不够应对可能的后续事件
3. **市场**：热度信号是上升还是下降，有没有AI竞品已经上线
4. **竞争**：提前上线能锁定空窗期用户，但低品质会被后来者蚕食

---

## 九、市场竞争结算

### 收益计算

**竞争范围：** 同题材内竞争，不同题材不争夺对方用户。

**两阶段分配：**

1. **空窗期用户（先到先得）**
   - 上线时无同题材竞品 → 这部分用户直接锁定
   - 提前上线的核心收益来源

2. **存量用户争夺（品质权重）**
   - 所有竞品上线后，剩余用户按品质权重分配
   - 高品质持续从低品质手里抢夺用户

**最终收益 = 空窗期用户 × 付费能力 + 存量份额 × 付费能力**

### 后端随机变量（每局随机生成）
| 变量 | 说明 |
|------|------|
| 各题材用户池大小 | 可争夺的用户总量 |
| 用户付费能力 | 平均付费水平 |
| AI竞品强度 | 竞品品质和激进程度 |
| 市场热度起点 | 各题材初始热度 |

### AI竞品行为
- 2-3个AI竞品参与同题材竞争
- 拟人化呈现（名字+性格标签），如"激进派总是抢先上线""保守派反复打磨"
- 行为可预测但不完全确定，给玩家信息解读空间

## 十、市场热度系统

### 呈现方式：模糊信号
- 不显示精确数值，显示趋势描述（如"热度攀升中""趋于饱和""突然爆发"）
- 玩家凭判断决策，不是计算最优解

### 动态变化驱动
1. **自然涨落周期** —— 每个题材有内在的热度曲线
2. **AI竞品上线消耗** —— 竞品上线后对应题材热度被分流
3. **偶发市场事件** —— 随机拉升或压低某题材（制造突发决策压力）

### 与收益的关系
- 热度作为乘数，品质作为基数
- 题材符合热度窗口有bonus，但不保证收益
- 市场总量固定，是零和博弈

## 十一、信息层设计原则

**所有外部信息只暴露行为结果，不暴露行为动机。玩家永远需要自己解读。**

| 信息类型 | 玩家看到的 | 玩家看不到的 |
|---------|-----------|------------|
| 市场热度 | 模糊趋势信号 | 精确数值和变化公式 |
| 自身品质 | 模糊等级（除非做内测） | 精确品质分 |
| AI竞品 | 上线时间、存在感 | 品质、策略、资源配置 |
| 事件选项 | 代价和收益的精确值 | 后续还会不会有更多事件 |

**设计意图：** 局部信息确定（单事件内可计算），全局信息不确定（整局形势需要判断）。决策能力 = 在不确定全局中利用确定局部做出最优选择。

---

## 十二、结算呈现

### 成功结算
- 逐条展示收益来源：空窗期用户、品质竞争份额、热度bonus
- 展示金钱收入，明确可以购买什么等级的入场资源

### 失败结算（强化失去恐惧）
- 逐条列出带入的每项资源及其购入价格
- 逐条标红"已损失"
- 不是一个"失败"弹窗，而是让玩家感受到每一分钱的消失
- 借鉴《竞拍之王》的失败呈现设计

## 十三、Demo 范围裁剪（4天红线）

### 必须实现（核心循环）
- [ ] 入场资源选购界面（3类×3级）
- [ ] 题材选择界面（3-4个题材+模糊热度信号）
- [ ] 研发流程主界面（时间条+品质模糊等级+事件触发）
- [ ] 随机事件弹出+选项选择（至少4-6个不同事件）
- [ ] 可选节点：内测验证（信息揭示）
- [ ] 可选节点：临上线打磨（风险增益）
- [ ] 上线按钮+确认
- [ ] 市场竞争结算逻辑+结果展示
- [ ] 失败结算逐条呈现
- [ ] 局间金钱传递+再入场

### 应该实现（数值手感）
- [ ] AI竞品拟人化（名字+性格标签+行为日志）
- [ ] 市场热度动态变化可视化
- [ ] 事件文本行业梗包装（至少每个事件有风格化文案）
- [ ] 音效反馈（时间流逝提示音、事件触发音、上线确认音）

### 可以砍掉（如果时间不够）
- [ ] 多局之间的难度递进
- [ ] 事件文本的多样化变体（每种事件只做1个文案够用）
- [ ] 精细的动画表现
- [ ] 教程/新手引导（资深制作人不需要）

### 不做
- 竞品动态信息系统（留给后续版本）
- 行业新闻事件系统
- PVP 模式
- 局外养成

## 十四、世界观约束

**禁止使用现实世界中的真实地名、产品名、品牌名、公司名。**

- 可以影射、借鉴现实原型，但必须使用虚构名称
- 命名风格：行业黑话+黑色幽默，让资深从业者会心一笑
- 已确定的影射命名：鹅皇互娱（腾讯）、天命堂（任天堂）
- 此规则适用于所有游戏内文本
