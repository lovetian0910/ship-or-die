# 模块04：研发流程与可选节点

> 拆分自：`2026-03-30-game-demo-full-spec.md`
> 模块职责：研发推进机制、品质积累、内测验证节点、临上线打磨节点

---

## 一、研发推进机制

- 选题后自动进入研发，时间开始流逝
- 品质随时间推进自动积累，速率受"时间效率"资源影响
- 品质积累有上限，受"品质上限"资源约束
- 品质对玩家呈现为**模糊等级**（如"粗糙/合格/精良/杰作"），不显示精确数值

### 品质模糊等级

| 内部品质分 | 显示等级 |
|-----------|---------|
| 0-25 | 粗糙 |
| 25-50 | 合格 |
| 50-75 | 精良 |
| 75-100 | 杰作 |

- 模糊呈现带误差：实际品质45分可能显示"合格"也可能显示"精良"（边界区域±10分随机偏移）
- 做内测后显示真实等级（去除偏移）

### 品质积累公式

```
品质增长/秒 = 基础速率 × 时间效率倍率
当前品质 = min(累计品质, 品质上限)
```

- 基础速率：约0.15分/秒（300秒总时长能达到约45分基础品质）
- 时间效率倍率：低1× / 中1.5× / 高2×
- 品质上限：低40 / 中70 / 高100

## 二、研发阶段划分

| 阶段 | 时间占比 | 时间范围（300秒局） | 事件密度 |
|------|---------|------------------|---------|
| 前期 | ~30% | 0-90秒 | 低：1固定探索 + 0-1被动危机 |
| 中期 | ~40% | 90-210秒 | 高：1固定探索 + 随机探索 + 1-2被动危机 + 内测节点 |
| 后期 | ~30% | 210-300秒 | 低：偶发随机探索 + 0-1被动危机 + 打磨节点 |

## 三、可选节点1：内测验证（中期，信息揭示型）

**时机：** 研发进度约50%时出现（约150秒），玩家可选择进入或跳过。

**机制：**
- 消耗一段固定时间（约30-45秒研发时间）
- 揭示当前品质的**真实等级**（平时品质显示是模糊的，带误差）
- 不改变品质本身，纯信息价值

**博弈点：**
- 时间充裕时：信息有价值，帮助后续决策
- 时间紧张时："花30秒只为看一个数字，值吗？"
- 与市场热度系统呼应：一个看外部（市场），一个看内部（自身品质），两个不完全信息拼合辅助上线决策

**实现要点：**
- 在进度约50%时弹出提示："是否进行内测验证？消耗30秒"
- 选择"是"：时间扣除，品质显示切换为真实等级（后续不再有偏移）
- 选择"否"：跳过，继续研发
- 节点只出现一次，错过不再出现

**设计意图（面试展示）：** 信息本身作为资源的设计思路——对应搜打撤中"搜"的本质不是直接获得战力，而是获得决策信息。

## 四、可选节点2：临上线打磨（后期，风险增益型）

**时机：** 研发进度约85%时出现（约255秒），紧贴上线决策窗口。

**机制：**
- 消耗一段时间（约30-45秒研发时间）
- 进入前明示两种可能结果及概率：
  - **70%概率：** 品质小幅提升（约1个模糊等级的1/3，即约8分）
  - **30%概率：** 发现严重bug，玩家面临二次选择：
    - 花15-20秒修复 → 品质不变（白花了打磨+修复的时间）
    - 放弃修复直接上线 → 品质反降1个等级（约25分）

**博弈点：**
- 做过内测的玩家：知道自己品质真实水平，能理性评估"小幅提升有没有意义"
- 没做内测的玩家：不知道自己品质到底够不够，只能凭感觉赌
- 时间剩余多：风险可控，值得试
- 时间剩余少：30%概率触发bug修复可能直接导致时间耗尽=全损

**实现要点：**
- 进度约85%时弹出提示，明示概率
- 选择"是"：执行随机判定，展示结果
- 如果触发bug：弹出二次选择（修复/放弃）
- 选择"否"：跳过

---

## 五、两节点咬合关系（核心设计亮点）

中期是否做内测，直接影响后期打磨的决策质量。这构成了一条完整的信息链路：
- 搜（内测）→ 知（品质真实水平）→ 判（打磨是否值得）→ 撤（上线时机）
- 不搜 → 不知 → 盲赌 → 可能失误

## 六、研发主界面需求

- 中央区域：研发进度可视化（时间条/进度环）
- 顶部：剩余时间 + 当前品质模糊等级
- 底部：上线按钮（始终可见）
- 侧边：市场热度信号面板
- 事件/节点以弹窗形式覆盖在研发界面上
- 竞品上线通知以toast形式出现
