## 一、上线决策（撤离等效）

这是整个游戏的**核心博弈时刻**。

### 机制

- 研发流程中玩家随时可以按"上线"按钮
- 按下后弹出确认对话框，确认后不可撤回，进入市场竞争结算
- 如果时间耗尽仍未上线 = 撤离失败，资源全损

### 决策维度

玩家在做上线决策时需要综合考量：
1. **品质**：当前品质够不够（如果做了内测就知道，没做就是模糊的）
2. **时间**：还剩多少时间，够不够应对可能的后续事件
3. **市场**：热度信号是上升还是下降，有没有AI竞品已经上线
4. **竞争**：提前上线能锁定空窗期用户，但低品质会被后来者蚕食

### 实现要点

- "上线"按钮始终显示在研发主界面底部
- 点击后弹出确认框，显示当前状态摘要：
  - 品质等级（模糊/真实，取决于是否做了内测）
  - 剩余时间
  - 市场热度信号
  - 已上线竞品数量
- 确认 → 进入结算；取消 → 返回研发

## 二、市场竞争结算

### 竞争范围

同题材内竞争，不同题材不争夺对方用户。

### 两阶段分配

**1. 空窗期用户（先到先得）**
- 上线时无同题材竞品 → 这部分用户直接锁定
- 提前上线的核心收益来源
- 从上线时刻到下一个竞品上线期间，独占市场

**2. 存量用户争夺（品质权重）**
- 所有竞品上线后，剩余用户按品质权重分配
- 高品质持续从低品质手里抢夺用户

### 收益公式

```
最终收益 = 空窗期用户 × 付费能力 + 存量份额 × 付费能力

空窗期用户 = 题材用户池 × 空窗期时长占比 × 热度乘数
存量份额 = 剩余用户池 × (自身品质 / 所有竞品品质总和)
付费能力 = 随机生成的题材参数
```

### 热度乘数

| 热度范围 | 乘数 |
|---------|------|
| 0-20 | 0.3× |
| 20-40 | 0.6× |
| 40-60 | 1.0× |
| 60-80 | 1.5× |
| 80-100 | 2.0× |

### 后端随机变量（每局随机生成）

| 变量 | 说明 | 参考范围 |
|------|------|---------|
| 各题材用户池大小 | 可争夺的用户总量 | 1000-5000 |
| 用户付费能力 | 平均付费水平 | 0.5-2.0 |
| AI竞品强度 | 竞品品质和激进程度 | 见模块03 |
| 市场热度起点 | 各题材初始热度 | 20-80 |

## 三、失败结算（撤离失败）

**时间耗尽未上线 = 全损**

### 呈现方式（强化失去恐惧）

- 逐条列出带入的每项资源及其购入价格
- 逐条标红"已损失"，带动画延迟（一项一项显示）
- 不是一个"失败"弹窗，而是让玩家感受到每一分钱的消失
- 借鉴《竞拍之王》的失败呈现设计

### 失败结算界面

```
┌─────────────────────────────┐
│      ❌ 开发超期，项目流产     │
│                              │
│  你带入了以下资源：            │
│                              │
│  🧑‍💻 鹅皇互娱主创  ¥300  [已损失] │  ← 逐条出现，标红
│  🏭 外包铁军      ¥300  [已损失] │
│  🤝 实习商务      ¥100  [已损失] │
│                              │
│  总计损失：¥700               │
│                              │
│  剩余金钱：¥XXX              │
│                              │
│  [再来一局]  [返回主菜单]      │
└─────────────────────────────┘
```
