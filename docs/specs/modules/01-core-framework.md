> 拆分自：`2026-03-30-game-demo-full-spec.md`
> 模块职责：游戏核心循环、时间机制、信息层设计原则

---

## 一、核心循环

```
入场（选购资源）
  ↓
选题（参考市场热度信号）
  ↓
研发流程（时间在流逝）
  ├─ 前期：固定探索点 + 0-1次被动危机
  ├─ 中期：固定探索点 + 随机探索点 + 1-2次被动危机 + 【内测验证节点】
  ├─ 后期：偶发随机探索点 + 0-1次被动危机 + 【临上线打磨节点】
  ↓
上线决策（撤离时刻）
  ↓
市场竞争结算
  ↓
带收益离场 / 撤离失败（资源全损）
```

### 单局时长：约10分钟

| 阶段 | 时长 | 说明 |
|------|------|------|
| 入场/选题 | 1-2分钟 | 选资源、选题材 |
| 研发流程 | 5-6分钟 | 核心游玩阶段，事件+探索+可选节点 |
| 上线+结算 | 2-3分钟 | 撤离决策+结果呈现 |

### 游戏状态机

核心循环对应以下状态流转，实现时以此为主状态机：

```
MENU → ENTRY_SHOP → TOPIC_SELECT → DEV_RUNNING → LAUNCH_CONFIRM → SETTLEMENT → (MENU 或 ENTRY_SHOP)
```

- `DEV_RUNNING` 是最复杂的状态，内部包含事件触发、可选节点、时间流逝等子状态
- `SETTLEMENT` 后根据结果进入下一局入场或返回主菜单

## 二、时间机制（生命值等效）

- 每局固定开发时间上限（Demo中建议300秒游戏内时间，加上事件决策的暂停时间，玩家实际体验约5-6分钟。UI上呈现为"开发周期"）
- 时间耗尽前未上线 = 撤离失败，入场资源全部损失
- 时间消耗来源：
  - 基础研发推进（自动消耗）
  - 探索节点（主动消耗）
  - 可选节点（主动消耗）
  - 危机事件选项（被动/主动消耗）
- 时间是唯一的衰减维度，不需要血量/饱食度等额外数值

### 实现要点

- 时间以秒为单位在后台tick，事件决策期间暂停倒计时
- UI呈现为进度条 + 剩余时间文字，后期变红闪烁提示紧迫感
- 时间耗尽时强制触发"撤离失败"结算

## 三、信息层设计原则

**所有外部信息只暴露行为结果，不暴露行为动机。玩家永远需要自己解读。**

| 信息类型 | 玩家看到的 | 玩家看不到的 |
|---------|-----------|------------|
| 市场热度 | 模糊趋势信号 | 精确数值和变化公式 |
| 自身品质 | 模糊等级（除非做内测） | 精确品质分 |
| AI竞品 | 上线时间、存在感 | 品质、策略、资源配置 |
| 事件选项 | 代价和收益的精确值 | 后续还会不会有更多事件 |

**设计意图：** 局部信息确定（单事件内可计算），全局信息不确定（整局形势需要判断）。决策能力 = 在不确定全局中利用确定局部做出最优选择。

## 四、搜打撤三要素映射

| 原型 | 设计映射 |
|------|--------|
| 搜 | 主动探索节点，消耗时间，固定+随机出现 |
| 打 | 研发流程中的随机事件应对，策略抉择 |
| 撤 | 主动决定上线时机，带收益离场 |
