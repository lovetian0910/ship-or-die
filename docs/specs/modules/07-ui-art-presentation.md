> 拆分自：`2026-03-30-game-demo-full-spec.md`

## 一、全局风格：像素风（Pixel Art）

- itch.io 免费/低价素材最丰富的风格
- 不同来源素材混搭违和感最低
- AI生成像素图比写实图更稳定
- 天然适合独立游戏题材的自我调侃

---

## 二、逐画面美术方案

| 画面 | 需要什么 | 来源 | 耗时 |
|------|---------|------|------|
| 主菜单/标题 | 氛围图1张 + logo文字 | AI生图 | 10分钟 |
| 入场选购 | 9个角色头像（3类×3级） | AI生图，统一prompt | 30分钟 |
| 选题界面 | 3-4个题材图标 | itch.io 图标素材包 | 10分钟 |
| 研发主界面 | 办公室背景 + UI框架 | AI生图1张背景 + itch.io UI包 | 20分钟 |
| 事件弹窗 | 事件配图6-8张 | AI生图，每事件一张 | 40分钟 |
| 代码急救小游戏 | IDE/终端风格 | 纯代码实现，零美术 | 0分钟 |
| 上线确认 | "发射按钮"仪式感画面 | AI生图1张 | 10分钟 |
| 结算画面 | 成功/失败氛围底图各1张 | AI生图 | 10分钟 |
| UI通用 | 按钮、面板、进度条、文字框 | itch.io 像素UI素材包1套 | 10分钟 |
| 字体 | 中文字体 | 免费开源（得意黑等） | 5分钟 |

**美术总投入：约2.5小时，零费用。**

### AI生图统一prompt模板

所有角色头像和事件配图使用统一风格前缀，保证画风一致：
> `pixel art, 32-bit style, dark background, game industry theme, [具体内容], clean lines, icon style`

### 降级方案

如果美术时间不够：
- 事件配图降级为"纯文字+角色头像复用"
- 背景图降级为纯色渐变+UI布局
- 角色头像降级为文字名称+色块头像框

## 三、世界观约束

**禁止使用现实世界中的真实地名、产品名、品牌名、公司名。**

- 可以影射、借鉴现实原型，但必须使用虚构名称
- 命名风格：行业黑话+黑色幽默，让资深从业者会心一笑
- 已确定的影射命名：鹅皇互娱（腾讯）、天命堂（任天堂）
- 此规则适用于所有游戏内文本

## 四、Demo 范围裁剪（4天红线）

### 必须实现（核心循环）

- [ ] 入场资源选购界面（3类×3级）
- [ ] 题材选择界面（3-4个题材+模糊热度信号）
- [ ] 自定义游戏名输入
- [ ] 研发流程主界面（时间条+品质模糊等级+事件触发）
- [ ] 随机事件弹出+选项选择（搜类事件，文字选项）
- [ ] "代码急救"小游戏（打类事件的gameplay实现）
- [ ] 可选节点：内测验证（信息揭示）
- [ ] 可选节点：临上线打磨（风险增益）
- [ ] 上线按钮+确认
- [ ] 市场竞争结算逻辑+结果展示
- [ ] 失败结算逐条呈现
- [ ] 局间金钱传递+再入场
- [ ] 基础美术素材接入（UI素材包+AI生图角色头像+背景）

### 应该实现（数值手感）

- [ ] AI竞品拟人化（名字+性格标签+行为日志）
- [ ] 市场热度动态变化可视化
- [ ] 事件文本行业梗包装（至少每个事件有风格化文案）
- [ ] 事件配图（AI生图）
- [ ] 音效反馈（时间流逝提示音、事件触发音、上线确认音）

### 可以砍掉（如果时间不够）

- [ ] 多局之间的难度递进
- [ ] 事件文本的多样化变体（每种事件只做1个文案够用）
- [ ] 精细的动画表现
- [ ] 教程/新手引导（资深制作人不需要）

### 不做

- 竞品动态信息系统（留给后续版本）
- 行业新闻事件系统
- PVP 模式
- 局外养成
